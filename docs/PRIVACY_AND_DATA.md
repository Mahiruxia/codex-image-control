# 隐私与数据边界

Image Control 是本机单人工作台。公开代码提供连接器框架，不提供维护者的视频生成服务。安装插件不会让其他使用者访问维护者自己的模型、工作流、账号或项目。

## 数据分类

| 数据 | 默认位置 | 是否进入发布包 | 建议 |
| --- | --- | --- | --- |
| 插件程序、技能、模板和离线说明 | Codex 插件缓存或源码仓库 | 是 | 可随版本替换，不在缓存中修改 |
| 项目 JSON、图片、视频 | `<有效状态根>\data\projects`，或 `IMAGE_CONTROL_PROJECTS_ROOT` | 否 | 单独备份，按敏感素材管理 |
| 最终连接器 profile、端点和工作流 | `<有效状态根>\data\local\video-providers` | 否 | 可能暴露内部服务信息，备份需加密 |
| 接入草稿与上传文件 | `<有效状态根>\data\local\video-provider-setups` | 否 | 不迁移、不备份，完成后及时清理 |
| API 密钥和密码 | Windows 凭据库 | 否 | 凭据库不可用时失败关闭；不写入项目、日志或发布包 |
| 临时队列、锁和能力文件 | `<有效状态根>\.runtime` | 否 | 不备份，可在完全退出应用后重建 |
| 构建产物 | `app\dist`、`runtime` 与 `artifacts` | 仅固定发布载荷 | `artifacts` 不提交；正式 ZIP 经过二次扫描 |

Windows 的有效状态根来自用户级 `IMAGE_CONTROL_STATE_ROOT`；未设置时为 `%LOCALAPPDATA%\CodexImageControl`。有效项目根来自用户级 `IMAGE_CONTROL_PROJECTS_ROOT`；未设置时为 `<有效状态根>\data\projects`。默认项目根嵌套在状态根中，因此删除整个状态根会同时删除默认项目。

自定义根目录只能指向明确的本机普通目录。不要使用磁盘根、用户主目录、源码仓库、插件安装缓存、公开云盘、UNC/多人共享目录、符号链接、目录联接或挂载点；不要让项目根与私有状态、运行目录、旧仓库或新仓库重叠。

## 对外发送

图片生成和改图由当前 Codex 任务的内置能力完成，其数据处理受所使用 Codex 服务的条款约束。

只有在使用者主动配置并提交视频任务时，插件才会把必要的输入图片、提示词和任务参数发送到该连接器的端点。使用者需要自行确认端点所有者、TLS、鉴权、保留期限、日志策略、费用和模型许可。公开插件没有默认维护者端点，也不会把未配置的视频任务转发到维护者服务。

连接器允许访问使用者自己的本机或局域网服务，因此 profile 和工作流属于受信任的本机配置。不要导入陌生人提供的配置；至少人工核对提交地址、状态地址、下载来源、工作流联网节点和凭据范围。

## 凭据处理

- UI 和工具响应只展示“是否已配置”等状态，不回传完整密钥。
- 错误、日志和连接测试必须清洗请求头、查询参数、Cookie、令牌和私有路径。
- scoped credential 绑定 profile ID 与端点、认证和下载范围；范围变化后旧凭据失效。
- 每个已经开始的 generic 视频任务会在内存中锁定同一份凭据快照，提交、轮询和下载不会混用任务开始后新换的账户。界面轮换影响新任务和崩溃恢复；已经开始的任务可能继续使用旧凭据直到结束或远端撤销。需要立即停止访问时，先在服务端撤销旧密钥。
- 旧版无 scope 的 legacy credential 不会自动绑定到迁移 profile，必须在本机 UI 重录。
- 删除仍然存在的单个连接器时，应用会删除该 profile 当前范围的凭据和同 ID 的 legacy credential。
- “彻底清除本插件保存的全部密钥”会在本机同源安全窗口中按服务 `image-control-video-provider` 枚举并清除当前项和已删除 profile 留下的孤儿项。
- 换机或重录凭据只使用连接器界面，不要手工构造 Windows 凭据库的内部账户名。
- 怀疑泄露时先在服务端撤销或轮换，再清理本机文件、历史、日志和截图。

同机恢复相同 profile ID 与相同安全范围时，已有 scoped credential 可能仍可用；这不是文件备份复制了密码，而是 Windows 凭据库独立保留了它。恢复后必须先审核端点和范围；不确定时清除后通过 UI 重录。

## 发布防泄露

`npm run scan:release` 会扫描源码、构建后的 JavaScript 和原生二进制可见字符串。正式打包还会对解压后的真实载荷再次扫描，并拒绝私有网络 URL、签名 URL、常见令牌、私钥、本机用户路径、工作流 JSON、用户媒体、本地数据目录和符号链接。

自动检测有边界：自定义编码、很短的秘密、图片里的文字、Git 历史、GitHub 日志和仓库外部系统仍需人工检查。公开前检查必须同时覆盖当前文件、历史、Issue、PR、Actions、截图和 Release 资产。

## 卸载与彻底清理

普通卸载只删除插件程序缓存，不删除外部状态、项目或 Windows 凭据。需要彻底清理时按以下顺序操作：

1. 在工作台视频模型列表点击“彻底清除本插件保存的全部密钥”，在本机安全窗口确认完成；如需撤销访问，同时在远端服务轮换或撤销密钥。
2. 备份需要保留的项目，并按 [备份与恢复](BACKUP_AND_RECOVERY.md) 验证数量、大小和 SHA-256。
3. 记录有效状态根和项目根，确认默认项目根是否嵌套在状态根内。
4. 完全关闭 Codex、独立工作台和其他使用这些目录的进程。
5. 从磁盘根到两个目标逐级确认没有重解析点；确认目标不是磁盘根、用户主目录、源码仓库或插件缓存，且没有意外的包含关系。
6. 先把精确目标重命名到同磁盘隔离目录，重新启动并确认不再需要，再删除隔离副本。默认项目根已随状态根一起隔离时，不要重复处理。
7. 删除用户级 `IMAGE_CONTROL_STATE_ROOT` 与 `IMAGE_CONTROL_PROJECTS_ROOT`，避免以后重装重新打开旧目录或误写到已删除路径。
8. 最后卸载插件。再次安装时应从空状态启动，且不会出现旧连接器或旧凭据。

清除用户级变量可在确认路径记录无误后执行：

```powershell
[Environment]::SetEnvironmentVariable("IMAGE_CONTROL_STATE_ROOT", $null, "User")
[Environment]::SetEnvironmentVariable("IMAGE_CONTROL_PROJECTS_ROOT", $null, "User")
```

命令执行后完全重启 Codex。它们只清除路径设置，不删除任何目录。

不要对用户主目录、桌面、文档目录、源码根或磁盘根执行递归删除。若在清理按钮发布前已经先删除了 profile 或状态，只能在 Windows 凭据管理器中谨慎查找服务标识 `image-control-video-provider` 的条目；不要删除无法确认归属的系统或其他应用凭据。
