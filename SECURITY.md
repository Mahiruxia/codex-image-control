# Security Policy

## 支持版本

安全修复优先覆盖最新正式版本。重大问题会在必要时回补最近一个次要版本。带 +codex. 后缀的本地缓存构建不是正式支持版本。

## 报告漏洞

请使用 GitHub 的私有漏洞报告功能，不要在公开 Issue、讨论、截图或日志中粘贴密钥、Cookie、私有视频接口、工作流、用户素材或可复现的敏感数据。

报告中请包含受影响版本、Windows 与 Node.js 版本、最小复现步骤、预期和实际结果。请先对所有地址、路径、请求头和素材脱敏。维护者确认后会协调修复、版本发布与披露时间。

如果已经公开泄露凭据，应先在对应服务端撤销或轮换它；仅删除 GitHub 文件或重写工作区并不能使已泄露的密钥失效。

## 信任与网络边界

Image Control 是单人、本机优先的工具，不是面向不可信多用户的托管服务。HTTP 工作台只应监听回环地址，并通过本机启动会话建立的授权与 CSRF 校验访问。不要把本地端口通过反向代理、端口映射、隧道或防火墙规则暴露到局域网或互联网。

视频连接器会向使用者明确配置的第三方或自建端点发送必要的图片、提示词和任务参数。该端点的运营者可能看到这些内容；插件无法替代使用者审核其隐私政策、日志策略和访问控制。公开版本没有默认的维护者视频端点。

连接器配置本身属于受信任的本机配置：本项目明确支持回环地址和局域网中的自建 ComfyUI，因此不会把所有私网目标一律拦截。不要导入或保存来源不明的连接器草稿、工作流或下载来源；保存前应核对最终主机、协议、认证范围与费用。插件不是可供不可信输入任意转发请求的网络沙箱或通用代理。

## 本地数据边界

- 项目图片、视频和项目 JSON 位于 IMAGE_CONTROL_PROJECTS_ROOT。
- 连接器资料和状态位于 IMAGE_CONTROL_STATE_ROOT；敏感凭据只进入 Windows 凭据库，凭据库不可用时失败关闭。
- .runtime 保存临时队列、锁和进程状态，可删除重建，不应同步或备份。
- 插件源码、安装缓存和正式发布包不得包含 data/local、data/projects、真实素材、私有工作流、Cookie、令牌或开发者电脑绝对路径。

备份时只复制明确选择的状态配置和项目目录。系统凭据库中的密码不会随文件备份迁移，换机后必须重新录入。详见 [备份与恢复](docs/BACKUP_AND_RECOVERY.md)。

## 发布安全门禁

npm run package:windows 会执行源码和打包后二次脱敏扫描、依赖高危漏洞审计、固定清单打包、符号链接和路径穿越检查、归档体积与压缩比限制、可复现哈希比对、SBOM 生成和运行时冒烟测试。GitHub Actions 均固定到完整提交 SHA，标签发布拒绝覆盖已有 Release。

扫描器会检查文本与原生二进制中的常见凭据格式、本机绝对路径、私有网络地址、签名 URL、私钥、用户媒体、私有工作流和本地数据目录。它无法保证发现所有自定义编码、短令牌、历史提交、外部日志或图片内可见信息，因此发布前仍需人工复核：

- 当前变更及完整 Git 历史。
- Issue、PR、Actions 日志和构建产物。
- README、截图、录屏、测试夹具和示例配置。
- 第三方许可证、NOTICE、SBOM 与依赖审计结果。

不要为了让扫描通过而把真实凭据加入允许列表。测试需要模拟敏感数据时，应在运行时由多个无害片段拼装，并继续断言系统会拒绝或清洗它。
