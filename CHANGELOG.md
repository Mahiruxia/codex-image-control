# Changelog

本项目遵循语义化版本。

## 0.11.1 - 2026-07-19

- MCP App 新增历史版本资源地址兼容层，旧 Codex 任务保存的 `workbench-v26.html` 或其他版本化地址会读取当前已安装工作台，同时继续使用精确版本 URI 刷新新任务缓存。
- 增加协议回归测试，固定现代与兼容资源元数据、MIME、旧地址读取和越界地址拒绝行为。
- 插件技能将 `render_workbench` 固定为唯一正常打开入口，禁止在资源错误时静默退回浏览器界面。
- Windows Release ZIP 改为标准 Codex 本地插件市场结构，包含市场清单、相对插件路径和离线安装说明，解压后可以直接登记市场并安装插件。
- 增加从 GitHub 仓库、固定版本标签及 Release ZIP 安装、升级和卸载的完整命令。

## 0.11.0 - 2026-07-19

- 公开版视频连接器改为每位使用者自行配置，移除任何维护者端点、工作流和凭据的隐式继承路径。
- 加固本机工作台的回环访问、会话授权、CSRF、防目录穿越、连接器 URL 与敏感错误信息处理。
- 视频任务新增请求级租约、提交状态机、连接器执行指纹与幂等恢复；远端是否已接收无法确认时默认停止，避免自动重提造成重复计费。
- 旧版无作用域凭据改为失败关闭，端点或认证范围变化即失效；新增一键清除本插件全部 Windows 凭据（含孤儿项）。
- 状态根、项目根与迁移流程增加真实路径、重解析点、目录重叠、活动视频任务和原子复制校验。
- 增加覆盖源码、构建产物和原生二进制的发布脱敏扫描，以及对应的失败关闭测试。
- 正式包新增 CycloneDX SBOM、第三方组件清单、上游 LICENSE/NOTICE、解包后二次扫描与 MCP 冒烟测试。
- Windows ZIP 使用固定顺序与时间戳，并通过双次构建 SHA-256 一致性验证可复现性。
- CI、CodeQL、依赖评审、Dependabot 与标签发布流程固定第三方 Action 的完整提交 SHA；已存在的 Release 不再允许覆盖。
- 根项目、插件、前端、服务端和锁文件统一升级至 0.11.0；本地缓存标记仅允许单个 +codex. 后缀，正式发布必须恢复纯基础版本。
- 新增长期维护、隐私边界、备份恢复、迁移与发布检查文档，并修正 TwitCanva 派生界面和本项目重写视频能力的归因描述。

## 0.10.1 - 2026-07-19

- 将插件程序、长期状态和用户项目目录彻底分离，升级安装不再覆盖本机私有配置。
- 新增 `IMAGE_CONTROL_STATE_ROOT`，Windows 默认使用 `%LOCALAPPDATA%\CodexImageControl`。
- 增加旧项目私有配置迁移脚本与长期维护说明。
- 健康检查、独立工作台启动器和测试同步验证状态目录，避免误复用旧版进程。
- 服务版本和工作台缓存 URI 自动跟随插件版本，并增加标签触发的 GitHub Release 与 SHA-256 校验文件。

## 0.10.0 - 2026-07-19

- 建立独立的开源维护仓库。
- 将前端、服务端、技能和模板收拢到自包含插件目录。
- 清除开发者电脑的绝对路径与私有项目目录。
- 增加 Windows 自包含运行包、版本同步、发布打包、持续集成和敏感文件检查。
